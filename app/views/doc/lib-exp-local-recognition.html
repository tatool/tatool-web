Experimental Tasks

Local Recognition

In this paradigm (modelled after Oberauer, 2005), participants have to memorise items and their current context (e.g., object at specific locations in a grid). After memorisation, participants are probed with matches (i.e., items that have been presented in that context), distractors (i.e., items that haven't been presented in the current trial at all), or lures (i.e., items that have been presented in a different context). This setup allows for disentangling recognition of items from recognition of item-to-context binding. Local recognition tasks are, therefore, sometimes also referred to as binding tasks (e.g., De Simoni & von Bastian, 2018; von Bastian & Oberauer, 2013).

Executable

Use the following settings in the Module Editor for this type of task:

Project tatool
Executable Type tatoolLocalRecognition

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
showKeys Boolean true No Shows or hides specified input keys.
displayDuration String 2000 No Timer duration in ms for display of a single memorandum.
intervalDuration String 2000 No Timer duration in ms for interval between memorandum.

Stimuli File

This task requires a stimuli file to run. The stimuli file has to contain the columns specified in the Stimuli File Structure below.

Available Stimuli Files

Task Project Type File Name Description
Local Recognition Grid tatool-stimuli stimuli localRecognition_grid.csv
localRecognition_grid_practice.csv		 Display of 2-5 coloured triangles shown in a 5x5 grid.
Local Recognition Row tatool-stimuli stimuli localRecognition_rows.csv
localRecognition_rows_practice.csv		 Display of 2-4 fractal images in a row of 2-4 columns.

Stimuli File Structure

Column Name Column Values Mandatory Column Description
stimulusCount (String) Yes Number of memoranda/probes for this trial.
gridRows (String) Yes Number of rows to show in grid.
gridCols (String) Yes Number of columns to show in grid.
stimulusValue[n] (String)/(Image Filename) Yes Value to display on screen. Can either be text or an image filename.
stimulusValueType[n] text/image/circle/square Yes The internal Tatool stimulus value type for Tatool to know how to display it.
stimulusGridPosition[n] (String) Yes The grid position for this memorandum.
probeValue[n] (String)/(Image Filename) Yes Value to display on screen. Can either be text or an image filename.
probeValueType[n] text/image/circle/square Yes The internal Tatool stimulus value type for Tatool to know how to display it.
probeGridPosition[n] (String) Yes The grid position for this probe.
probeType[n] match/intrusion/distractor Yes The type of this probe.
match: probe matches item and position.
intrusion: probe matches item but not position.
distractor: probe matches neither item nor position.
keyCount (String) No Number of keys to be added for this stimuli (by default 2; mandatory for more than 2 keys).
keyCode[n] (KeyCode) Yes The correct keyCode to press from the list of available KeyCodes (see Tatool Input Service).
keyLabel[n] (String)/(Image Filename) No An alternative label to display on the input key. Can either be text or an image filename.
keyLabelType[n] text/image No The internal Tatool key label type for Tatool to know how to display it.
response[n] (String) Yes Response value assigned to this key.
