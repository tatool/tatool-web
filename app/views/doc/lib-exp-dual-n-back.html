Experimental Tasks

Dual N Back

In the Dual N-Back paradigm, participants need to respond whenever the current stimulus was the same as the one presented N positions back in the sequence. Dependent measure is the proportion of hits minus false alarms averaged over both modalities and all n-back levels

Element

Use a List Element as the parent element in order to create the Dual N-Back paradigm. The number of iterations should be set to the number of stimuli to be shown.

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
displayDuration String 2000 No The time in milliseconds to display each stimulus.
nValue String 1 No The critical number of positions back in the sequence.
iterations String 20 No The number of iterations as provided in the List Element. Used to assist in generating the correct number of targets.
targets String 12 No The number of targets to be generated.

Stimuli File

ldual-nback-permutations.csv contains a list of the available visual and audio stimuli.
