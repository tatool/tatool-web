Experimental Tasks

Monitoring

In this paradigm (modelled after Oberauer, Süß Wilhelm, & Wittman, 2003), participants have to monitor constantly changing elements and react whenever a critical relation between those elements occurs (e.g., three rhyming words appear in a row, column, or diagonal of a matrix).

Executable

Use the following settings in the Module Editor for this type of task:

Project tatool
Executable Type tatoolMonitoring

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
showKeys Boolean true No Shows or hides specified input keys.
displayDuration String 2000 No Timer duration in ms for display of a single trial.
gridRows String 3 No Number of rows to show in grid.
gridCols String 3 No Number of columns to show in grid.
cellHeight String 150 No Height of each cell.
cellWidth String 150 No Width of each cell.

Stimuli File

This task requires a stimuli file to run. The stimuli file has to contain the columns specified in the Stimuli File Structure below.

Available Stimuli Files

Task Project Type File Name Description
Squares Monitoring tatool-stimuli stimuli luco_monitoring_fig.csv
luco_monitoring_fig_practice.csv		 Monitor changing dot patterns in a 10x10 grid to detect a square.
Digits Monitoring tatool-stimuli stimuli luco_wm_num.csv
luco_wm_num_practice.csv		 Monitor changing numbers in a 3x3 grid to detect when the last digits of 3 numbers in a row, column or on the diagonal are identical.

Stimuli File Structure

Column Name Column Values Mandatory Column Description
stimulusCount (String) Yes Number of filled cells in this trial.
stimulusValue[n] (String)/(Image Filename) Yes Value to display on screen. Can be either text or an image filename.
stimulusValueType[n] text/image/circle/square Yes The internal Tatool stimulus value type for Tatool to know how to display it.
gridPosition[n] (String) Yes The grid position for this item.
gridCellClass[n] (String)/(Image Filename) Yes Styling of this cell. Can be either tatoolMonitoring_bgBlackCell for a black background (e.g., to highlight specific cells), or tatoolMonitoring_bgWhiteCell for a white background (e.g., when displaying text).
stimulusType (String) Yes Condition of this trial, can be used to keep track of trials and runs.
correctResponse (String) Yes Correct response of this trial.
keyCode (KeyCode) Yes The keyCode to press from the list of available KeyCodes (see Tatool Input Service) whenever the critical relation occurs.
keyLabel (String)/(Image Filename) No An alternative label to display on the input key. Can either be text or an image filename.
keyLabelType text/image No The internal Tatool key label type for Tatool to know how to display it.
response (String) Yes Response value assigned to this key.

Example Module

Download from GitHub: demoMonitoring.json

The Example Module contains all elements of the demo and can be used to customize the task (for more details, see instructions for using your own stimuli and description of the Instruction Accessory Executable). Note that the Instruction Executable currently doesn't allow HTML pages from external resources. This means that you can use custom HTML pages if you host your own instance of Tatool Web, but you cannot use externally hosted HTML pages on tatool-web.com. For running experiments on tatool-web.com with customized instructions, you must use images (see the FAQ for currently available options to host your custom stimuli and instructions on tatool-web.com).
